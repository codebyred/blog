MikroTik VLANs: "Interface" vs. "Bridge" FilteringIf you are diving into MikroTik’s RouterOS, you will quickly notice there are two different places to configure VLANs. Choosing the wrong one can lead to poor performance or a complete network lockout. Here is the definitive guide on when and why to use each.1. Interface > VLAN (The Layer 3 Virtual Port)Creating a VLAN under the Interface menu is the equivalent of creating a Virtual Sub-Interface.What it is: A logical port that sits on top of a physical interface (like ether1).The Purpose: To give the router a "foot" in a specific VLAN so it can perform Routing, provide DHCP, or apply Firewall rules.When to use it: Use this when you need the MikroTik to act as the Gateway for that VLAN.Key Rule: If you need to assign an IP address to a VLAN, you create it here.2. Bridge > VLAN (The Layer 2 Managed Switch)The Bridge menu is where you define how VLAN tags move between ports. This is called VLAN Filtering.What it is: A way to turn your MikroTik ports into "Access" ports (untagged) or "Trunk" ports (tagged).The Purpose: To pass traffic across the device at high speeds, ideally using Hardware Offloading so the CPU doesn't break a sweat.When to use it: Use this to define your VLAN table—which ports belong to which VLAN ID.The ComparisonFeatureInterface > VLANBridge > VLAN (Filtering)Primary GoalInter-VLAN RoutingVLAN Switching/TrunkingLogic"The Router needs an IP here""Traffic from Port A goes to Port B"PerformanceCPU-bound (Slower)Hardware-offloadable (Faster)Network RoleDefault GatewayManaged SwitchThe "Best Practice" WorkflowIn modern RouterOS (v6.41+), you rarely attach a VLAN interface directly to a physical port. Instead, follow this standard workflow:Create a Bridge: Add your physical ports (ether1, ether2, etc.) to a single bridge.Define VLANs: In Bridge > VLANs, specify which ports are Tagged (Trunks to other switches) and Untagged (Access ports for PCs).Create the Interface: In Interfaces > VLAN, create the VLAN interface but set its Parent Interface to your Bridge.Assign IP: Go to IP > Addresses and assign your gateway IP to the VLAN interface you just created.Enable Filtering: Finally, go to the Bridge settings and set vlan-filtering=yes.Pro-Tip: The Golden RuleDon't cross the streams. If a port is a "Slave" (part of a bridge), you should generally not attach an Interface > VLAN directly to that physical port. Always attach your Layer 3 VLAN interfaces to the Bridge to ensure your traffic follows the correct hardware path!
